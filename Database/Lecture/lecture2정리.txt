관계 데이터 모델이 큰 성공을 거둔 요인
- 바탕이 되는 데이터 구조, 간단한 테이블(릴레이션)을 사용함.
- 중첩된 복잡한 데이터 구조 없음
- 집합 위주의 데이터 처리
- 이론이 잘 정립
- 효율적인 질의 처리
- 좋은 성능
- 쉽게 이해


동일한 구조(릴레이션)의 관점에서 모든 데이터를 논리적으로 구성
: 논리적인 연결을 위해 링크나 포인터를 사용 안 함.

선언적인 질의어를 통한 데이터 접근을 제공
: 사용자는 원하는 데이터만 명시, 어떻게 찾는 지는 명시 X
: 응용 프로그램들은 데이터 베이스 내의 레코드들의 어떠한 순서와도 무관함


릴레이션
            애트리뷰트
         ---------------------------
레코드   |
=튜플    |
        |
        |

애트리뷰트      => 내연
레코드, 튜플    => 외연

내연은 변화 잘 없음
외연은 계속해서 변화함

도메인:
한 애트리뷰트 내에 나타날 수 있는 값들의 집합
애트리뷰트의 도메인 값들은 원자값
ex 데이터 타입값
복합 애트리뷰트나 다치 애트리뷰트는 허용되지 않음


차수와 카디널리티

차수: 한 릴레이션에 들어있는 애트리뷰트들의 수
    유효한 릴레이션의 최소 차수는 1
    릴레이션의 차수는 변하지 않음

카디널리티: 릴레이션의 튜플 수
    유효한 릴레이션은 카디널리티 0값을 가질 수 있음
    릴레이션의 카디날리티는 시간이 지남에 따라 계속해서 변함
